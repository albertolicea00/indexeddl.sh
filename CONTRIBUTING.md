# Contributing Guidelines

Thank you for considering contributing to **indexeddl.sh**! We value community contributions and strive to make the process as smooth as possible.

## How to Contribute

1. **Fork the repository** on GitHub by clicking the *Fork* button on [indexeddl.sh](https://github.com/albertolicea00/indexeddl.sh).
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/<your-username>/indexeddl.sh.git
   cd indexeddl.sh
   ```
3. **Add the upstream remote** so you can keep your fork in sync:
   ```bash
   git remote add upstream https://github.com/albertolicea00/indexeddl.sh.git
   ```
4. **Create a new branch** for your feature or bug‑fix:
   ```bash
   git checkout -b my-feature
   ```
5. **Make your changes** following the project's coding style.
6. **Commit your changes** with a clear, concise message following [Conventional Commits](https://www.conventionalcommits.org/).
7. **Push to your fork** and **open a Pull Request** targeting the `main` branch of the upstream repository.

## Pull Request Checklist

- [ ] Follows the project style guidelines.
- [ ] Includes relevant documentation updates.
- [ ] Passes all existing tests.
- [ ] Adds new tests for new functionality (if applicable).
- [ ] References any related issues (e.g., `Closes #123`).

## Reporting Issues

If you encounter a problem, please open an issue using the provided [Issue Template](.github/ISSUE_TEMPLATE.md).

We appreciate your help in making indexeddl.sh better!

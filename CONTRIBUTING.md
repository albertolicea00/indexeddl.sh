# Contributing Guidelines

Thank you for considering contributing to **indexeddl.sh**! We value community contributions and strive to make the process as smooth as possible.

## How to Contribute
1. **Fork the repository** and clone your fork locally.
2. **Create a new branch** for your feature or bug‑fix:
   ```bash
   git checkout -b my-feature
   ```
3. **Make your changes** following the project's coding style.
4. **Run tests** (if applicable) and ensure everything passes.
5. **Commit your changes** with a clear, concise message.
6. **Open a Pull Request** targeting the `main` branch of the upstream repository.

## Pull Request Checklist
- [ ] Follows the project style guidelines.
- [ ] Includes relevant documentation updates.
- [ ] Passes all existing tests.
- [ ] Adds new tests for new functionality (if applicable).
- [ ] References any related issues (e.g., `Closes #123`).

## Code Style
- Use **snake_case** for variable names and **PascalCase** for class names.
- Keep line lengths under 100 characters.
- Include descriptive comments where necessary.

## Reporting Issues
If you encounter a problem, please open an issue using the provided **Issue Template**.

We appreciate your help in making indexeddl.sh better!
